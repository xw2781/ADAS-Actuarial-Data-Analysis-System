# Release 0.2.1

Released on 2026-04-28.

## User-Facing Changes

### Improvements
- **scripting**: Organized scripting console frontend files into a dedicated UI folder.
  - The scripting console page and companion JavaScript modules now live together under ui/scripting_console.

### Fixes
- **startup**: Restored the ArcRho logo on the startup splash screen.
  - The splash screen now references the shared icon asset from the correct relative path.

## Internal Notes

- **build**: Added automated release fragment validation and per-version release note generation.
  - Successful packaged builds now archive unreleased fragments into versioned release notes.
  - Contributors can record changes in small JSON fragments instead of editing one large changelog by hand.
- **developer workflow**: Added a reusable agent commit-and-push helper.
  - Agents now have a guarded root-level PowerShell workflow for staging, committing with a conversation-specific message, validating the GitHub remote, and pushing the active branch.
  - The local repository bootstrap now uses a single ArcRho root Git repository with ignore rules for dependencies, bundled toolchains, build outputs, caches, and editor state.
- **app server**: Renamed the frontend server package to app_server.
  - The FastAPI package now lives under frontend/app_server, with imports, packaging entry points, and generated documentation paths updated to match.
- **frontend messaging**: Renamed the frontend cross-frame message namespace from arcrho to arcrho.
  - Electron menu relays, shell iframe coordination, feature-page messages, pop-out channels, and documentation now use arcrho:* message names.
